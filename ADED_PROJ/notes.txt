Large language models based on the transformer architecture have become central to modern natural language processing 
systems. These models rely on attention mechanisms and large matrix multiplications to process input sequences and 
generate outputs. While GPUs are typically used to accelerate these computations, there is increasing interest in 
running such models on CPU-only systems due to cost, availability, or deployment constraints.

CPUs differ significantly from GPUs in terms of architecture. They generally offer fewer cores but more sophisticated 
control logic, deeper cache hierarchies, and access to larger system memory. However, they also have lower memory 
bandwidth and reduced parallel compute throughput compared to GPUs. As a result, running large transformer models 
efficiently on CPUs requires careful optimization.

One of the primary challenges is the model’s memory footprint. Large models can require several gigabytes of memory 
to store weights, and during inference, these weights must be accessed repeatedly. This can lead to memory bandwidth 
bottlenecks, especially when the working set does not fit within CPU caches. Additionally, cache locality becomes 
critical, as poor data layout can result in frequent cache misses and degraded performance.

Another important factor is the autoregressive nature of text generation. During decoding, tokens are generated 
sequentially, which limits the amount of parallelism that can be exploited. Although some parallelism is possible 
within each forward pass, the dependency between tokens introduces a fundamentally sequential component that 
impacts throughput and latency.

Thread-level parallelism is also a key consideration. CPUs can use multiple threads to accelerate computation, 
but increasing the number of threads does not always lead to linear speedups. Contention for shared resources 
such as caches and memory bandwidth can reduce efficiency. Techniques such as thread pinning and NUMA-aware memory 
allocation are often required to achieve optimal performance.

Quantization has emerged as a practical solution to some of these challenges. By reducing the precision of model 
weights (for example, from 32-bit floating point to 8-bit or 4-bit integers), quantization can significantly
reduce memory usage and improve cache efficiency. This can also lead to faster computation due to reduced data 
movement. However, quantization may introduce numerical errors that affect model accuracy, particularly in 
sensitive components such as attention layers.

Based on this context, what are the main challenges of running large transformer models on CPU-only systems? 
How does quantization help address these challenges, and what trade-offs does it introduce in terms of performance 
and model quality?
